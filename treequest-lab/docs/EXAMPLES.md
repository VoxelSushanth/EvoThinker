# Example Experiments

This document provides concrete examples of experiments you can run with TreeQuest Lab.

## Example 1: Basic Architecture Search

**Goal**: Find better activation functions for small CNNs on CIFAR-10.

```bash
treequest run \
  --prompt "Discover novel activation functions that improve convergence speed for small CNNs on image classification" \
  --max-nodes 10 \
  --benchmark cifar10_subset \
  --config configs/architecture_search.yaml
```

**Expected Output**:
- Tree with 10 nodes exploring different activation combinations
- Best hypothesis might be: "Swish activation with learnable beta parameter"
- Report showing accuracy improvements over baseline ReLU

## Example 2: Optimizer Discovery

**Goal**: Explore novel optimizer combinations or learning rate schedules.

```bash
treequest run \
  --prompt "Find learning rate schedules that converge faster than cosine annealing for transformer training" \
  --max-depth 4 \
  --max-width 5 \
  --model-name gpt-4o-mini \
  --dry-run  # Test first without execution
```

## Example 3: Data Augmentation Search

**Goal**: Discover effective data augmentation strategies.

```python
# scripts/run_augmentation_search.py
from treequest import Tree, Config

config = Config(
    root_prompt="Novel data augmentation techniques for low-data regimes",
    max_nodes=15,
    benchmark="cifar10_10percent",
    domain="computer_vision"
)

tree = Tree(config)
tree.run()
tree.generate_report()
```

## Example 4: Ablation Study - Tree Search vs Random

Compare MCTS against random search:

```bash
# Run MCTS
treequest run --max-nodes 20 --strategy mcts --seed 42 --output-dir runs/mcts_run

# Run random baseline
treequest run --max-nodes 20 --strategy random --seed 42 --output-dir runs/random_run

# Compare
python scripts/compare_runs.py runs/mcts_run runs/random_run
```

## Example 5: Multi-Stage Refinement

Run iterative refinement with increasing budgets:

```bash
# Stage 1: Quick exploration (low budget)
treequest run --max-nodes 5 --timeout 60 --output-dir stage1

# Stage 2: Expand promising branches
treequest resume stage1 --max-nodes 15 --timeout 180 --output-dir stage2

# Stage 3: Deep dive on best path
treequest resume stage2 --max-depth 6 --timeout 300 --focus-best --output-dir final
```

## Tips for Better Results

1. **Start Specific**: Narrow prompts work better than broad ones
   - Good: "Improve convergence speed for ResNet-18 on CIFAR-10"
   - Bad: "Make ML models better"

2. **Set Constraints**: Define clear resource limits
   ```yaml
   execution:
     max_time_per_experiment: 120  # seconds
     max_memory_mb: 1024
   ```

3. **Seed Diversity**: Run multiple seeds for robust conclusions
   ```bash
   for seed in 42 43 44; do
     treequest run --seed $seed --output-dir run_$seed
   done
   ```

4. **Monitor Progress**: Use WandB integration
   ```bash
   export WANDB_PROJECT=treequest-experiments
   treequest run --track-wandb
   ```

5. **Save Checkpoints**: Long runs can be resumed
   ```bash
   treequest run --checkpoint-every 5  # Save every 5 nodes
   ```

## Common Patterns

### Pattern 1: Hypothesis Refinement

```
Root: "Improve transformer efficiency"
├─ Child 1: "Use sparse attention" → Score: 0.6
│  └─ Grandchild: "Local window attention" → Score: 0.8
├─ Child 2: "Reduce parameters" → Score: 0.5
└─ Child 3: "Better initialization" → Score: 0.7
   └─ Grandchild: "Orthogonal init + LayerNorm" → Score: 0.85
```

### Pattern 2: Parallel Exploration

```
Root: "Data augmentation search"
├─ Branch A: Geometric transforms (rotate, flip, crop)
├─ Branch B: Color space augmentations
├─ Branch C: Mixup/CutMix variants
└─ Branch D: Neural augmentation (AutoAugment style)
```

### Pattern 3: Failure Recovery

When experiments fail, Critic agent learns:

```
Failed: "Learning rate 1.0" → Reflection: "Too high, causes divergence"
Next proposal: "Learning rate 0.1 with warmup" → Success!
```
